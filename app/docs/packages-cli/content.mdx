# @paljs/cli (Deprecated)

> **`@paljs/cli` has been removed in PalJS v9.** Code generation is now handled by `@paljs/generator` as a native Prisma generator.

## Migration

Replace `@paljs/cli` with the new generator workflow:

**Before (v8):**

```bash
npm install -g @paljs/cli
pal generate
```

**After (v9):**

```bash
npm install -D @paljs/generator
npx prisma generate
```

Add the generator to your `schema.prisma`:

```prisma
generator paljs {
  provider = "paljs-generator"
  output   = "./generated/paljs"
}
```

Replace `pal.config.js` with `paljs.config.ts`:

```typescript
import { defineConfig } from '@paljs/generator/config';

export default defineConfig({
  generateGraphQL: true,
  generateTypes: true,
});
```

## What Replaced Each Command

| Old Command | Replacement |
|---|---|
| `pal create` | Use `create-next-app` or `create-t3-app`, then add PalJS packages |
| `pal generate` | `prisma generate` (with `paljs` generator in schema) |
| `pal generate admin` | `generateAdmin: true` in `paljs.config.ts` |
| `pal schema json` | Use `@paljs/generator` DMMF writer output |
| `pal schema camel-case` | No longer supported |
